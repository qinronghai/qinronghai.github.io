关于JavaScript中的this

AHAI
2023-09-16
JavaScript, 学习笔记

在JavaScript中，this是一个特殊的关键字，它代表当前执行代码的上下文对象。this的值在不同的情况下会有所不同，取决于代码的执行环境。

绑定规则

默认绑定

如果函数独立调用，且没有使用严格模式，this会绑定到全局对象（在浏览器环境中是window对象，在Node.js环境中是global对象）。这是最常见的情况。

严格模式下this是undefined

隐式绑定

当函数作为对象的方法被调用时，this会绑定到调用该方法的对象。

显式绑定

可以使用call()、apply()或bind()方法显式地指定函数执行时的this值。

apply-应用的意思

call-调起的意思

call()和apply()是JavaScript中的两个函数方法，它们用于显式地调用函数，并指定函数执行时的this值。

这两个方法的作用是相同的，它们的区别在于参数的传递方式：

call()方法接受一个指定的this值，后面可以跟随多个参数，每个参数都是作为函数的参数传递进去。

apply()方法也接受一个指定的this值，但是只接受两个参数：一个是作为函数的参数传递进去的对象，另一个是一个数组，其中包含要传递给函数的参数。

使用call()和apply()方法可以改变函数执行时的this值，使函数在不同的上下文中执行。这对于借用其他对象的方法、实现继承等场景非常有用。此外，call()和apply()还可以用于调用内置函数，并将类似数组的对象转换为真正的数组。

需要注意的是，如果传递的this值为null或undefined，在非严格模式下，它们会被自动替换为全局对象。而在严格模式下，this的值将保持为null或undefined

bind-绑定的意思

为什么要使用bind，那是因为apply和call在使用时，只是调用一次绑定执行一次。而bind可以绑定一次，后面调用时this总是指向绑定的obj对象身上。

bind()方法用于创建一个新函数，新函数会将指定的this值绑定到原函数，并返回这个新函数。

new绑定

当函数作为构造函数使用时，this会绑定到通过构造函数创建的实例对象。

内置函数的调用绑定

有时候，我们会调用一些javascript的内置函数，或者一些第三方库中的内置函数，这些内置函数，往往需要我们传入另外一个函数如（setTimeout、forEach）。这些内置函数是由js内部或者第三方库进行调用的，并不是我们决定什么时候调用，所以很多时候我们并不知道这些函数中的this是如何绑定的，这就需要根据一些经验才知道

this绑定的规则优先级比较

学习了四条this的绑定规则之后，只需要知道应用了哪条规则即可。但是如果一个函数调用位置应用了多条规则，这时候就出现了优先级比较的问题。

  1. 默认调用规则优先级最低
  2. 显示绑定的优先级高于隐式绑定
  3. new绑定优先级高于隐式绑定
  4. new绑定优先级高于bind、new绑定和call、apply是不允许同时使用的，所以不存在谁的优先级更高的问题、
  5. bind的优先级高于apply和call

总结：从高到低

  1. new
  2. bind
  3. apply和call
  4. 隐式绑定
  5. 默认绑定

  1. 如果在显示绑定中，传入一个null或者undefined，那么这个显示绑定会被忽略，使用默认规则。
  2. 间接函数引用
