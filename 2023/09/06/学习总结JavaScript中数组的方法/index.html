    slice/splice 方法

    slice

    arr.slice([start], [end]) 返回一个原数组的从索引 start 到 end（不包括 end）浅拷贝副本，是一个新数组（有新的内存地址）。

    arr.slice() 则会创建一个原数组完整的副本，同样是一个有着原数组相同元素的新数组。如果对新数组进行修改，原数组不会受到影响。

    数组（引用类型）的浅拷贝

    数组浅拷贝是指在原数组的基础上创建一个新的数组，复制了原数组的引用（即内存地址）。
    所以，当修改拷贝后的数组时，原数组也一样会发生改变。

    像上面的这种直接赋值的方式就是数组的浅拷贝，浅拷贝改变其中一个数组，另外一个数组也会跟着改变。很多时候，这不是我们想要的。

    这就需要通过上面的slice方法满足我们的需求：拷贝出来的数组中某项被修改后，对原数组没有影响。

    splice

    splice是一个强大的处理数组的方法，它可以“就地/原地”：添加，删除和替换元素。

    arr.splice(start[, deleteCount, elem1, ..., elemN])

    split和join方法

    split是一个字符串类型的方法，通过给定的分隔符将字符串分割成一个数组。

    split(separator[, limit])

    join是反过来的split，通常配合split。

    concat

    concat可以将其他数组/项连接起来，返回一个新的数组。

    操作数组（难点）

    map方法

    map方法是最有用和经常使用的方法之一，它创建一个新数组，并对每个元素都调用函数，结果数组是由每次调用给定函数的返回值组成。

    reduce/reduceRight方法

    reduce方法是一个根据数组计算单个值，返回一个计算后的值。

    该函数一个接一个地应用于所有数组元素，并将其结果“搬运（carry on）”到下一个调用。

    参数：

    accumulator —— 是上一个函数调用的结果，第一次等于 initial（如果提供了 initial 的话）。
    item —— 当前的数组元素。
    index —— 当前索引。
    arr —— 数组本身。
    应用函数时，上一个函数调用的结果将作为第一个参数传递给下一个函数。

    因此，第一个参数本质上是累加器，用于存储所有先前执行的组合结果。最后，它成为 reduce 的结果。

    [initial]是初始值，为accumulator第一次的值。如果没有initial，reduce会将数组的第一个元素作为初始值，并从第二个元素开始迭代。

    reduceRight只是从右到左的顺序。

    some/every方法

    这两个方法都对数字的每个元素调用函数fn，some相当于||运算符，如果fn返回一个真值，arr.some()立即返回true，并停止迭代其余数组项。

    every相当于&&运算符，如果fn返回一个假值，arr.every()立即返回false，并停止迭代其余数组项。只有所有数组项调用的fn返回真值时才会返回true。

    fill方法

    arr.fill()方法用一个固定值填充数字从start索引到end索引内的全部元素。返回修改后的数组。

    fill(value, start, end)

    数组的排序

    sort/reverse

    对数组进行原位排序，更改元素的顺序，即修改了原数组。
